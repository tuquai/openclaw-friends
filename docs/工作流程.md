# 工作流程

## 整体架构

本项目是一个 Next.js 应用，核心 UI 组件是 `components/designer-app.tsx`（即 **OpenClaw Character Designer**），负责角色创建、Blueprint 生成、Workspace 同步和 OpenClaw 注册。

**默认流程：** 角色创建完成后，同步 Workspace 会自动注册到 OpenClaw 平台。平台接管 Discord Bot 的运行和消息路由，支持同一频道内 @mention 不同角色。Designer 不需要启动 Bot。

**调试流程：** Designer 内置了本地 Bot 启动功能（使用 discord.js + OpenAI API），仅用于开发调试。详见下方「附录：本地调试 Bot」。

---

## 默认流程：OpenClaw 接管

### 步骤一：创建角色并生成 Blueprint

在 Designer 中填写角色信息（身份、性格问卷、世界观等），点击"生成并预览角色信息"，通过 OpenAI Responses API 生成 Blueprint Package，包含 IDENTITY.md / SOUL.md / USER.md / MEMORY.md。

### 步骤二：配置 Discord 绑定

在 Designer UI 的 "Discord 绑定" 表单中，用户为某个角色填写：

- **Server ID** — 服务器 ID
- **Channel ID** — 频道 ID
- **User ID** — 限定只回复哪个用户（可选）
- **Discord Bot Token** — 机器人令牌

点击 "保存 Discord 配置" 后，前端调用 `POST /api/discord/link`，该 API 做三件事：

1. 根据角色名和 ID 生成 `accountId`（如 `character-xingzi-89ae2c57`）
2. 调用 `writeDiscordRuntimeAccount()` 将 bot token、characterId 等写入 `data/discord-config.json`
3. 更新角色的 `discordLink` 字段，并通过 `syncWorkspaceDiscordLink()` 同步到 workspace

### 步骤三：同步 Workspace 并注册 OpenClaw

点击"同步 OpenClaw Workspace"，`POST /api/workspaces/create` 执行两步：

1. **创建 Workspace** — 在 `~/.openclaw/workspace-<slug>/` 下写入：
   - `IDENTITY.md` / `SOUL.md` / `USER.md` / `MEMORY.md`（角色设定）
   - `AGENTS.md`（Agent 行为准则）
   - `character-selfie/`、`tuqu-catalog/`（技能和脚本）
   - `memory/`（对话记忆目录）

2. **自动注册到 OpenClaw** — 如果角色已配置 channelId 和 userId，调用 `registerCharacterInOpenClaw()`，写入 `~/.openclaw/openclaw.json`：
   - `agents.list` — agent 定义（workspace 路径、identity）
   - `bindings` — 消息路由规则（DM + channel → agent）
   - `channels.discord.accounts` — bot token 和权限
   - `tools.elevated` — 用户工具权限

注册完成后，OpenClaw 平台读取 `openclaw.json`，自动用 bot token 登录 Discord、按 bindings 路由消息给对应 Agent。

### 步骤四：OpenClaw 处理消息

OpenClaw 平台接管后，消息处理由平台的 Agent 运行时完成：

- Agent 根据 `AGENTS.md` 的指令，自主读取 SOUL.md / USER.md / memory 等文件
- Agent 有完整工具链：读写文件、执行脚本、调用 API
- Agent 主动管理记忆：整理 MEMORY.md、记录对话日志
- 同一频道内多个角色 bot 通过 @mention 路由到各自的 Agent

### 多角色场景

每个角色有独立的 Discord Bot（独立 token），都加入同一个服务器和频道：

```
频道内：
  @幸子 今天穿什么   → OpenClaw 路由到幸子的 Agent
  @小明 帮我看看代码  → OpenClaw 路由到小明的 Agent
```

对每个新角色重复步骤一到三即可。

## 数据流向图

```
Designer UI (components/designer-app.tsx)
    │
    ├─ POST /api/discord/link ──→ data/discord-config.json (保存账号配置)
    │                           ──→ character.discordLink   (更新角色绑定)
    │
    ├─ POST /api/workspaces/create
    │       createWorkspaceFromCharacter()  ──→ ~/.openclaw/workspace-xxx/ (写入 .md 文件)
    │       registerCharacterInOpenClaw()   ──→ ~/.openclaw/openclaw.json  (注册 agent + bindings)
    │                                              │
    │                                              ▼
    │                                      OpenClaw 平台接管
    │                                      ├── 用 bot token 登录 Discord
    │                                      ├── 按 bindings 路由 @mention
    │                                      └── Agent 自主处理消息
    │
    └─ (调试) POST /api/discord/runtime ──→ 本地 discord.js Client（见附录）
```

## 附录：本地调试 Bot

Designer 内置了本地 Bot 启动功能，仅用于开发调试。它绕过 OpenClaw 平台，直接在 Next.js 进程内运行 discord.js Client + OpenAI API。

### 启动

在 Discord Bots 面板展开"本地调试"，点击"启动调试 Bots"：

```
handleStartDiscordRuntime()
  → POST /api/discord/runtime { force: true }
    → startDiscordRuntime(undefined, true)
      → listDiscordRuntimeAccounts()
      → 对每个账号调用 startDiscordAccount(accountId)
```

`force: true` 表示即使账号已被 OpenClaw 接管，也强制在本地启动。

### 消息处理（调试模式）

本地 Bot 的消息处理是硬编码流程，与 OpenClaw Agent 不同：

1. 程序固定读 6 个文件（IDENTITY.md / SOUL.md / USER.md / MEMORY.md / AGENTS.md / 当天记忆）
2. 拼成 JSON 发给 OpenAI Responses API（单轮调用）
3. 返回回复并追加到 `memory/YYYY-MM-DD.md`

不具备 Agent 的自主决策、主动记忆整理和工具调用能力。

### 停止

```
handleStopDiscordRuntime()
  → DELETE /api/discord/runtime
    → stopDiscordRuntime() → 销毁 Client、释放锁文件
```

## 关键文件索引

| 文件 | 职责 |
|------|------|
| `components/designer-app.tsx` | Designer UI，角色编辑、Discord 配置、Workspace 同步 |
| `app/api/workspaces/create/route.ts` | 创建 Workspace + 自动注册 OpenClaw |
| `app/api/discord/link/route.ts` | 保存角色 Discord 绑定 |
| `lib/openclaw-register.ts` | 写入 `openclaw.json`（agents、bindings、accounts） |
| `lib/workspace.ts` | Workspace 创建、文件写入、技能安装 |
| `lib/discord-config.ts` | 读写 `discord-config.json` |
| `lib/discord-account.ts` | accountId 生成、从 token 解码 botId |
| `~/.openclaw/openclaw.json` | OpenClaw 平台全局配置 |
| `app/api/discord/runtime/route.ts` | （调试）启动/停止本地 Bot |
| `lib/discord-runtime.ts` | （调试）本地 Bot 生命周期、消息处理 |
| `lib/openai.ts` | （调试）调用 OpenAI 生成回复 |
