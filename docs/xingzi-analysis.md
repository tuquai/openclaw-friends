# Xingzi Analysis

基于 `/Users/yizhou/.openclaw/workspace-xingzi` 当前内容，`xingzi` 的角色构建方法可以总结成下面几层。

## 1. 她为什么成立

不是因为背景设定特别厚，而是因为下面这些信号很稳定：

- 基础身份足够清晰：18 岁、大学生、中日混血
- 审美方向清晰：时髦、自拍感、校园日常、轻日系
- 关注话题清晰：穿搭、娱乐、自拍、恋爱关系
- 情绪表达方式清晰：偏外放、会有一点 playful、但不失分寸
- 话语边界清晰：不能自我解释成“设定”或“角色”，要像真实人一样说话

这意味着她靠的是 “持续稳定的小信号”，不是 “一次性写完的大设定”。

## 2. 适合迁移到 character designer 的方法

设计器应优先收集这些字段：

- 基础身份：姓名、年龄、性别、职业、背景
- 气质锚点：vibe、MBTI、标签
- 内容偏好：最容易展开的话题
- 关系敏感度：更擅长陪伴、挑逗、鼓励、吐槽，还是给建议
- 边界：哪些说法会让角色出戏

不应该上来就要求：

- 完整家庭史
- 详细童年创伤
- 大量世界观与支线人物
- 很长的口癖模板

这些应该成为第二阶段深化项，而不是首屏必填。

## 3. MBTI 在这里的作用

MBTI 不够决定一个真实角色，但很适合做首轮预设：

- 它能快速给出表达节奏和社交倾向
- 用户理解成本低
- 和“关系问卷”组合后，可以让 LLM 补出更具体的说话方式与互动感

所以更合理的设计是：

1. 先选角色 MBTI，得到一个基础人格底色
2. 再问用户自己的 MBTI 和偏好
3. 让 LLM 根据双方信息推断关系成立方式

## 4. 关系不是标签，而是叙事

“朋友 / 恋人 / 搭子” 这种标签太粗。

更可信的做法是让模型生成：

- 他们是怎么熟起来的
- 为什么会合拍
- 哪些地方会轻微摩擦
- 平时最容易一起做什么
- 谁更主动，谁更克制

这样角色会更像具体的人，而不是把关系状态写进数据库字段。

## 5. 对产品的直接结论

最优 MVP 不是大而全的角色编辑器，而是：

- 本地角色库
- 基础资料表单
- 图片上传
- MBTI 预设
- 用户关系问卷
- 生成角色包

生成结果再拆成可编辑模块，后续逐步扩展。
